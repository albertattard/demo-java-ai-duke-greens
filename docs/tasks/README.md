# Delivery tasks

This directory contains small, implementation-ready units of delivery work. Each task links to the specification or architecture decision that defines its outcome, identifies its visitor value (or explains why it is enabling work), and lists completion and verification criteria.

## Contents

- [`TASK-0001-establish-application-baseline.md`](TASK-0001-establish-application-baseline.md) establishes the executable Java application and automated-test baseline.
- [`TASK-0002-present-welcome-and-request-input.md`](TASK-0002-present-welcome-and-request-input.md) lets a visitor begin the MVP journey.
- [`TASK-0003-present-authoritative-product-catalogue-on-landing-page.md`](TASK-0003-present-authoritative-product-catalogue-on-landing-page.md) presents the local product catalogue as landing-page cards.
